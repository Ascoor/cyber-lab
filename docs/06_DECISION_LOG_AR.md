# سجل القرارات المعمارية

## القرار 001: استخدام FastAPI

### السبب
- مناسب لبناء APIs بشكل سريع وواضح.
- يدعم Swagger UI تلقائيًا.
- يسهل اختبار endpoints محليًا.
- مناسب للمرحلة الحالية داخل WSL2 Ubuntu.

### الأثر
- سهولة اختبار `/` و`/health`.
- قابلية توسع لاحقة لإضافة endpoints للفحوصات والتقارير.
- الحاجة إلى validation واضح داخل endpoints قبل تشغيل أي فحص.

## القرار 002: منع shell=True

### السبب
- تقليل خطر command injection.
- منع تنفيذ أوامر غير مقصودة أو مدخلات مستخدم خطيرة.
- جعل تشغيل الأدوات الخارجية قابلًا للمراجعة.

### الأثر
- تشغيل الأدوات يجب أن يكون عبر قوائم أوامر آمنة.
- يجب عدم تمرير flags من المستخدم مباشرة.
- كل أداة خارجية تحتاج timeout وerror handling.

## القرار 003: البداية بفحوصات محدودة

### السبب
- تقليل المخاطر القانونية والتقنية.
- بناء أساس قابل للمراجعة قبل التوسع.
- ضمان أن المشروع يبقى دفاعيًا ومصرحًا.

### الأثر
- لا يتم إضافة فحوصات واسعة أو هجومية في المراحل الأولى.
- Target Management وسياسة النطاق تأتي قبل التوسع.
- أي موديول جديد يحتاج مواصفة واختبارات قبول.

## القرار 004: اعتبار التوثيق العربي مرجع التطوير الأساسي

### السبب
- توحيد منهجية العمل داخل المشروع.
- تسهيل مراجعة التغييرات من مطورين آخرين.
- جعل النطاق القانوني والأمني واضحًا قبل التنفيذ.

### الأثر
- يجب تحديث `docs/05_CHANGELOG_AR.md` بعد كل تعديل مهم.
- يجب تحديث `docs/06_DECISION_LOG_AR.md` عند وجود قرار معماري جديد.
- يجب الرجوع إلى Roadmap وSecurity Policy قبل إضافة أي ميزة.
