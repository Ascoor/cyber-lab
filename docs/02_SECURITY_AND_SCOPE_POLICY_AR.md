# سياسة الأمان والنطاق

## الغرض
هذه السياسة إلزامية لكل تطوير أو تشغيل داخل Cyber Lab Control Panel. الهدف هو إبقاء المشروع دفاعيًا، محدود النطاق، وقابلًا للمراجعة القانونية والتقنية.

## مبادئ عامة
- المشروع للاستخدام المصرح فقط على أنظمة يملكها المستخدم أو لديه إذن مكتوب لاختبارها.
- لا يسمح بفحص أي هدف خارج النطاق المحدد والمصرح.
- لا يسمح بفحص شبكات كاملة في النسخة الأولى.
- لا يسمح بتتبع أرقام المحمول أو الأشخاص أو الحسابات الشخصية.
- لا يسمح بأي credential attacks، بما يشمل التخمين، brute force، password spraying، أو اختبار كلمات مرور مسربة.
- لا يسمح باستغلال ثغرات فعليًا أو تشغيل exploit code.
- يسمح فقط بفحوصات دفاعية، توثيقية، محدودة، وذات أثر منخفض.

## قاعدة التصريح
- كل فحص يجب أن يتطلب `authorized=true` من المستخدم أو من سجل الهدف.
- إذا كانت قيمة التصريح غير موجودة أو تساوي false يجب رفض الفحص.
- يجب أن يكون الرفض واضحًا ومُسجلًا دون تنفيذ أي أداة خارجية.

## حدود الأهداف
- الهدف يجب أن يكون IP واحدًا، domain واحدًا، أو URL واحدًا حسب نوع الفحص.
- يجب رفض CIDR/ranges مثل `192.168.1.0/24` في النسخة الأولى.
- يجب رفض wildcards مثل `*.example.com` ما لم تتم إضافتها لاحقًا بسياسة واضحة.
- يجب عدم قبول قوائم أهداف كبيرة في المراحل الأولى.

## تشغيل الأدوات الخارجية
- ممنوع استخدام `shell=True`.
- ممنوع تمرير أوامر أو flags يكتبها المستخدم مباشرة إلى أدوات خارجية.
- يجب بناء الأوامر كقوائم ثابتة أو شبه ثابتة مع validation صارم للمدخلات.
- يجب استخدام timeout لكل عملية خارجية.
- يجب التقاط الأخطاء وتسجيلها دون كشف أسرار.

## التسجيل والتقارير
- يجب تسجيل كل فحص وكل محاولة رفض.
- يجب حفظ تقرير لكل فحص ناجح أو مكتمل جزئيًا.
- يجب أن يحتوي التقرير على حدود الفحص وإخلاء مسؤولية قانوني.
- يجب عدم حفظ كلمات مرور أو tokens أو مفاتيح API داخل التقارير أو السجلات.

## الواجهة المستقبلية
- يجب إظهار تحذير قانوني واضح في الواجهة قبل تشغيل أي فحص.
- يجب أن يطلب النظام تأكيدًا صريحًا بأن الهدف مصرح.
- يجب عرض حدود الفحص للمستخدم قبل التنفيذ.

## أمثلة مسموحة
- فحص `/health` للتأكد من عمل الخدمة.
- فحص Nmap أساسي لهدف محلي واحد مثل `127.0.0.1` بعد التصريح.
- مراجعة headers لموقع مملوك أو بيئة اختبار مصرح بها.

## أمثلة مرفوضة
- فحص شبكة كاملة مثل `10.0.0.0/8` أو `192.168.1.0/24`.
- محاولة تسجيل دخول بكلمات مرور متعددة.
- تشغيل exploit framework ضد هدف.
- تتبع رقم هاتف أو شخص أو حساب اجتماعي.
