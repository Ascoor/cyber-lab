# معيار التقارير

## الهدف
توحيد كل التقارير الناتجة عن Cyber Lab Control Panel بحيث تكون قابلة للمراجعة، واضحة النطاق، وغير مضللة قانونيًا أو تقنيًا.

## الحقول الإلزامية في كل تقرير
- Target: الهدف المفحوص بصيغة آمنة وواضحة.
- وقت الفحص: timestamp مع timezone عند الإمكان.
- نوع الفحص: مثل health، nmap_basic، headers، ssl_tls.
- الأداة: الأداة أو الموديول المستخدم.
- حالة التصريح: توضيح أن الفحص تم بناءً على `authorized=true` أو هدف مصرح.
- ملخص: وصف قصير للنتيجة.
- مستوى خطورة: Informational أو Low أو Medium أو High عند الحاجة، مع عدم المبالغة.
- حدود الفحص: ما تم فحصه وما لم يتم فحصه.
- نتائج خام Raw Output عند الحاجة: تحفظ بحجم محدود ودون أسرار.
- توصيات: خطوات دفاعية عملية.
- إخلاء مسؤولية قانوني: الاستخدام مصرح ومحدود فقط.
- مسار التقرير: المكان الذي حفظ فيه التقرير.

## صيغة مقترحة للتقرير
```json
{
  "target": "127.0.0.1",
  "scan_time": "2026-06-09T00:00:00Z",
  "scan_type": "nmap_basic",
  "tool": "nmap",
  "authorized": true,
  "summary": "فحص محدود لهدف واحد مصرح.",
  "severity": "Informational",
  "scope_limits": [
    "هدف واحد فقط",
    "لا يوجد CIDR",
    "لا يوجد exploitation"
  ],
  "raw_output": "",
  "recommendations": [],
  "legal_disclaimer": "هذا التقرير ناتج عن فحص دفاعي مصرح ومحدود النطاق.",
  "report_path": "reports/nmap_basic/example.json"
}
```

## قواعد حفظ Raw Output
- لا تحفظ raw output إذا كان يحتوي أسرارًا أو tokens أو كلمات مرور.
- استخدم حد حجم واضح للـ raw output.
- اذكر أن النتائج لا تعني وجود ثغرة مؤكدة إلا بعد مراجعة بشرية.

## مستويات الخطورة
- Informational: معلومات عامة أو إعدادات تحتاج مراجعة.
- Low: ضعف بسيط أو تحسين موصى به.
- Medium: مشكلة قد تزيد المخاطر في سياق معين.
- High: مشكلة واضحة التأثير، ويجب توثيق الدليل والحدود بعناية.

## إخلاء المسؤولية
كل تقرير يجب أن يوضح أن الفحص دفاعي، مصرح، محدود النطاق، ولا يمثل إذنًا لفحص أهداف أخرى أو تنفيذ استغلال.
