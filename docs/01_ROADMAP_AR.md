### مخرجات المرحلة
- توثيق عربي عملي.
- سياسة نطاق وأمان إلزامية.

- افتراض وجود وظائف غير منفذة.

### ما الذي لا يجب فعله في هذه المرحلة

- لا تغير أسماء endpoints الحالية.
- لا تضف scan واسع أو exploitation.

## المرحلة 1: Target Management

### الهدف
إضافة إدارة أهداف مصرح بها قبل أي فحص، بحيث لا يسمح النظام بفحص هدف غير مسجل أو غير مؤكد التصريح.

### الملفات المتوقعة
- `backend/app/models.py`
- `backend/app/database.py`
- `backend/app/auth.py`
- `backend/app/main.py`
- ملفات توثيق محدثة داخل `docs`.

### مخرجات المرحلة
- نموذج Target يحتوي الاسم، النوع، العنوان، حالة التصريح، وحدود الفحص.
- endpoints لإضافة وعرض الأهداف المصرح بها.
- validation لمنع ranges غير مسموحة في النسخة الأولى.

### شروط القبول
- لا يمكن تنفيذ فحص بدون `authorized=true`.
- يتم رفض CIDR/ranges عند الحاجة.
- يتم تسجيل كل محاولة إضافة أو تعديل هدف.

### مخاطر المرحلة
- قبول مدخلات غير واضحة مثل نطاقات واسعة.
- عدم التمييز بين IP وdomain وURL.

### ما الذي لا يجب فعله في هذه المرحلة
- لا تضف scanning فعليًا جديدًا.
- لا تسمح باستيراد قوائم أهداف كبيرة.

## المرحلة 2: Nmap Basic Scan مضبوط

### الهدف
تنفيذ فحص Nmap أساسي ومحدود ومصرح، بدون shell injection، وبدون scan واسع.

### الملفات المتوقعة
- `backend/app/modules/nmap_scan.py`
- `backend/app/main.py`
- `backend/app/reports/report_builder.py`
- `docs/07_TESTING_AND_VALIDATION_AR.md`

### مخرجات المرحلة
- endpoint لفحص هدف واحد فقط.
- خيارات Nmap محدودة وآمنة مثل `-sV` أو فحص منافذ محددة عند الحاجة.
- timeout واضح.
- تقرير محفوظ لكل فحص.

### شروط القبول
- رفض `authorized=false`.
- رفض CIDR مثل `192.168.1.0/24`.
- عدم وجود `shell=True`.
- وجود timeout ومعالجة أخطاء.
- حفظ تقرير بنتيجة الفحص.

### مخاطر المرحلة
- توسيع نطاق الفحص دون قصد.
- تمرير خيارات Nmap من المستخدم مباشرة.
- تشغيل فحوصات بطيئة أو مزعجة.

### ما الذي لا يجب فعله في هذه المرحلة
- لا تسمح بفحص شبكات كاملة.
- لا تسمح بخيارات NSE هجومية.
- لا تضف scripts استغلال.

## المرحلة 3: Web Security Baseline

### الهدف
إضافة فحوصات ويب دفاعية أساسية مثل توفر الموقع، معلومات HTTP العامة، وأخطاء إعداد ظاهرة بدون استغلال.

### الملفات المتوقعة
- `backend/app/modules/headers_scan.py`
- `backend/app/modules/zap_scan.py` عند الاستخدام المحدود لاحقًا
- `backend/app/main.py`
- `docs/08_MODULE_SPECIFICATION_TEMPLATE_AR.md`

### مخرجات المرحلة
- فحص URL واحد مصرح.
- نتائج baseline واضحة وغير هجومية.
- توصيات عامة قابلة للتنفيذ.

### شروط القبول
- لا يوجد crawling واسع.
- لا يوجد active attack scan.
- timeout واضح.
- تقرير محفوظ.

### مخاطر المرحلة
- تشغيل DAST نشط بدون ضبط.
- إرسال عدد طلبات كبير.

### ما الذي لا يجب فعله في هذه المرحلة
- لا تضف brute force.
- لا تضف fuzzing هجومي.
- لا تفعل ZAP Active Scan افتراضيًا.
