# قالب مواصفة موديول فحص جديد

استخدم هذا القالب قبل إضافة أي موديول جديد. لا يبدأ التنفيذ قبل إكمال الحقول الأساسية ومراجعتها مقابل سياسة الأمان.

## اسم الموديول
اكتب اسمًا واضحًا مثل: `nmap_basic_scan`.

## الغرض
اشرح لماذا نحتاج هذا الموديول وما القيمة الدفاعية منه.

## نوع الفحص
مثال: قراءة فقط، تحقق إعدادات، فحص منافذ محدود، تحليل static، مراجعة headers.

## هل يحتاج تصريح؟
نعم. كل موديول فحص يجب أن يتطلب تصريحًا صريحًا مثل `authorized=true` أو هدفًا مسجلًا كمصرح.

## اختبارات القبول
- حالة `authorized=true` لهدف محلي أو مصرح.
- حالة `authorized=false` مرفوضة.
- CIDR/range مرفوض.
- target غير صالح مرفوض.
- timeout يعمل.
- report محفوظ.

## مخاطر محتملة
- توسع نطاق غير مقصود.
- raw output يحتوي بيانات حساسة.
- استهلاك موارد.
- نتائج false positives أو false negatives.

## ما الذي لا يفعله هذا الموديول
- لا ينفذ استغلال ثغرات.
- لا ينفذ brute force.
- لا يفحص شبكات كاملة.
- لا يتتبع أشخاصًا أو حسابات.
